# ADR-001: Target Frequency for `mac_unit` (500 MHz, generic 28 nm)

- Date: 2026-04-24
- Status: ACCEPTED (Phase 3, REQ-U-001)
- Deciders: rtl-architect, timing-advisor

## Context

REQ-P-003 was a P1 PROVISIONAL requirement (500 MHz default), and OPEN-1-004 /
OPEN-2-001 deferred the final commitment to P3. P3 must commit a final number
so that `timing_constraints.json` and the SDC strategy can be produced. The
2-stage pipeline has ample margin at 500 MHz on any reasonable 28 nm or better
node; the decision mainly affects the `target_library` and clock uncertainty
choice.

## Options Considered

### Option A: Confirm 500 MHz on generic 28 nm cell library
- **Pros**: Conservative; meets REQ-P-001/P-002 trivially; no need for fast-adder
  cells; no DSP primitive requirements; SDC is simple (`-period 2.000`).
- **Cons**: Leaves performance on the table for advanced-node deployments.
- **Impact**: 5-register design, no re-pipelining required.

### Option B: Raise target to 1 GHz on advanced node
- **Pros**: Higher peak throughput per pin if a faster product cycle exists.
- **Cons**: Stage-2 33-bit add becomes the critical path; may require carry-
  lookahead or fast-adder cells; SDC complexity rises; no project requirement
  justifies this jump; would require library/timing data not available.
- **Impact**: Possibly an extra pipeline stage to stay 2-cycle-latency-clean.

### Option C: FPGA target
- **Pros**: Different deployment market.
- **Cons**: Different tool flow (DSP48 inference); not in current project scope.
- **Impact**: Tool-chain rework outside Phase 3.

## Decision

**Option A — 500 MHz on generic 28 nm cell library.**

Rationale:
- Confirms REQ-P-003 verbatim.
- No timing_constraints.json or library data exists in this environment to
  justify a higher commit.
- Architecture margin is ample (2 ns >> 33-bit add propagation on 28 nm).

## Consequences

**Positive:**
- Phase 4 RTL can use the literal `clk` port at 500 MHz with no special primitives.
- Phase 5 synthesis target is unambiguous: `create_clock -period 2.000`.
- No carry-lookahead, no DSP primitive, no re-pipelining.

**Negative:**
- If a downstream project demands >500 MHz, Phase 3 must be re-opened to
  decide whether to add a Stage-2 split or use a faster adder topology.

**Trade-offs:**
- Performance ceiling capped at 500 MMAC/s (well above any expressed user need).
