# ADR-004: Unit Testbench Framework (cocotb)

- Date: 2026-04-24
- Status: ACCEPTED (Phase 3, REQ-U-004)
- Deciders: uarch-designer, bfm-dev, ref-model-dev

## Context

The project supports cocotb (Python) and SystemVerilog testbenches. `mac_unit`
is small (5 registers, 9 ports) and combinatorially simple, so either is
viable; the choice affects P4/P5 effort and reuse. OPEN-2-004 defers to P3.

## Options Considered

### Option A: cocotb (Python) with Python reference model wrapping refc/
- **Pros**: Fastest iteration time. Constrained-random easy in Python.
  refc/mac_unit.c can be wrapped via cffi (or re-implemented bitexact in
  pure Python for 51-bit state). Project standard. Good cocotb-coverage tooling.
- **Cons**: Python adds runtime overhead vs. SV TB (negligible at this design size).
- **Impact**: Single Python testbench file in `sim/cocotb/mac_unit/`.

### Option B: SystemVerilog testbench with DPI-C reference model
- **Pros**: Matches larger-project convention. Direct DPI-C call into refc/.
- **Cons**: Heavier setup (compile DPI library, link into TB). Slower
  iteration than Python. Less flexible constrained-random.
- **Impact**: SV TB plus Makefile wiring for DPI.

### Option C: Both frameworks (cocotb smoke + SV regression)
- **Pros**: Highest coverage and broadest tool compatibility.
- **Cons**: Duplicated maintenance for a 5-flop design. Marginal value.
- **Impact**: 2x test maintenance burden.

## Decision

**Option A — cocotb (Python) with Python ref model wrapping refc/.**

Rationale:
- Fastest end-to-end for a single-module unit at this scale.
- refc/mac_unit.c is reusable as the golden reference (via cffi or
  bitexact Python re-implementation).
- Project already supports cocotb in standard makefiles.
- Constrained-random and corner-case generation are first-class in Python.

## Consequences

**Positive:**
- P4 wave-9 (unit test) authoring is fast.
- Phase 5 cocotb regression directly produces the Requirement Traceability
  Matrix in the Python toolchain.

**Negative:**
- A SystemVerilog TB would have been required if the project standard
  changed mid-flight; revisiting would mean a P3/P4 change request.

**Trade-offs:**
- None significant for a single-module design.
